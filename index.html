What It Does

💬

Real-Time Messaging

Messages broadcast instantly to all users in the same channel via Socket.IO room events — no polling or page reloads.

📢

Multi-Channel

Users create named channels and switch between them. Each channel maintains its own message history capped at 100 messages.

🗑️

Message Deletion

Users can delete their own messages. Deletions broadcast to all channel members and remove instantly from every client's view.

⚡

No Framework

The entire UI is built with a hand-rolled el() component system — functionally equivalent to React.createElement but in ~30 lines.

Socket.IO Events

-- Client → Server
login    { username }                       → restore session, return channels
join     { username, channel }             → join Socket.IO room, return history
open     { channel }                       → create channel, broadcast to all
send     { username, message }             → append + broadcast to room
delete   { username, message, timestamp }  → remove + broadcast to room
leave    { username }                      → leave room
logout   { username }                      → remove user, clear localStorage

-- In-memory storage: channels dict → list of messages (max 100)
